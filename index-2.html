System Design

Dual-Layer Architecture

The framework fuses a physical mobility layer (GPS convoy trajectories, Sybil attacker nodes) with a cyber layer (CybORG v2 network simulation) into a unified Gym-compatible environment. Blue DQN agents act independently per defender host; Red agents follow an adaptive finite-state machine.

📡
Physical Layer (VANET)
GPS-based convoy simulation with YAML-defined subnetworks. Nodes join/leave subnets as vehicles move. Sybil attackers inject fake presences at configurable density. Proximity threshold gates cyber engagement windows.
physical_layer.py PhysicalLayer v13.1 pandas · numpy
🔴
Red Agent — AttackFSM
Finite-state adversary cycling through Discover → Exploit → Escalate → Impact. Attack rate decays smoothly after neutralisation via rolling window (50-step). Configurable 1–4 simultaneous red agents with epsilon-greedy exploration.
attack_fsm.py cyber_layer.py v17.8
🛡️
Blue Agents — DQN
Independent DQN per defender host. Actions: Monitor, Analyse, Remove, Restore, Sleep. Prioritised Experience Replay · Huber loss · LR warmup (200 steps) · gradient clipping 5.0. Rewards clipped ±20 to prevent Q-value explosion.
rl_agent.py v13.5 PyTorch PER buffer
⚡
Reward & Observation
Composite reward: cyber success + physical proximity + resilience bonus. Observation builder flattens host compromise state, session info, and VANET subnetwork membership into a normalised vector per agent.

rl_agent.py — DQN core (v13.5) 
# Independent DQN agent with Prioritised Experience Replay
class DQNAgent:
    def __init__(self, state_dim, action_dim, lr=3e-4,
                 gamma=0.99, warmup_steps=200):
        self.q_net    = DQNetwork(state_dim, action_dim)
        self.target   = DQNetwork(state_dim, action_dim)
        self.buffer   = PrioritisedReplayBuffer(capacity=50_000)
        self.loss_fn  = F.smooth_l1_loss  # Huber — stable with large rewards

    def learn(self):
        states, actions, rewards, next_states, dones, weights =             self.buffer.sample(self.batch_size)
        rewards = rewards.clamp(-20, 20)  # prevent Q explosion

        # Double DQN target
        with torch.no_grad():
            next_actions = self.q_net(next_states).argmax(1)
            q_target = self.target(next_states).gather(1,
                           next_actions.unsqueeze(1))
            td_target = rewards + self.gamma * q_target * (~dones)

        loss = self.loss_fn(q_pred, td_target, reduction='none')
        (loss * weights).mean().backward()
        nn.utils.clip_grad_norm_(self.q_net.parameters(), 5.0)
cyber_layer.py — Rolling attack-rate window (v17.8) 
# Smooth attack-rate decay after Red neutralisation
def _compute_atk_rate(self):
    if not self.active_reds:
        self._atk_window.append((0, 0))   # slide old entries out
    total_att = sum(a for a, _ in self._atk_window)
    total_suc = sum(s for _, s in self._atk_window)
    if total_att == 0:
        return 0.0
    return total_suc / total_att   # decays over ~50 steps
